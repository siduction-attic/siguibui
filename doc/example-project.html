1. Introduction

This howto document describes the development of an application using the sigui toolkit.

The example is a simple application which allowes the user to do something with network tools.

2. Directory/File Structure

The project use some files from the toolkit. This files/directories will be inserted by symbolic links.

progensibu
	index.php (link to siguibui/index.php)
	backend
		progensibu.sh
	base (link to siguibui/base)
	config
		progensibu.conf
		progensibu_de.conf
		progensibu.css
		progensibu.html
	debian
		install
	etc
		progensibu
			progensibu.conf
		pywwetha
			progensibu.conf
			virtualhosts.d
				progensibu
	images
		favicon.ico
		logo.png
	plugins
		homepage.php
		home.content.txt
	usr
		bin
			progensibu

The 4 files in config will be copies of the configuration files from siguibui/build. This files will be extended while the development.

sidu-net-center.sh, favicon.ico, logo.png and homepage.php are new files.

2.1.File Modifications

  • standard.html: the name of the css file has changed to "sidu-net-center.css"
  • sidu-net-center.conf: All texts and configuration data will be put into this files. The language is english.
  • sidu-net-center_de.conf: This is a copy of sidu-net-server.conf with translated texts (german)
  • sidu-net-center.css: the cascaded style sheets forms the design of the page.

2.2.New Files

  • favicon.ico: This icon is shown in the address line of the browser.
  • logo.png ist the logo of the application.
  • sidu-net-center_de.conf: This is a copy of sidu-net-server.conf with translated texts (german)

2.2.1.homepage.php

This file is a plugin. Therefore it must contain a class which is derived from Page.

The following methods must be implemented:

  • __construct(): contains the initializations.
  • build(): Loads the content template.
  • onButtonClick(): handles the button clicks.

2.2.1.home.content.html

This file contains the html code of the content area.

  • __construct(): contains the initializations.
  • build(): Loads the content template.
  • onButtonClick(): handles the button clicks.
